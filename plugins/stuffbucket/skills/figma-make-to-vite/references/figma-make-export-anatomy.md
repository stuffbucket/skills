# Figma Make Export Anatomy

A reference for every file and directory in a Figma Make ZIP export, based on
observed real-world exports. Use this when reading or patching a Figma Make
project.

---

## Root files

| File | Purpose |
| ------ | --------- |
| `index.html` | Vite HTML entry. Mounts `<div id="root">` and loads `src/main.tsx` as an ES module. Contains the app `<title>` (set to the Figma frame/project name). |
| `package.json` | All dependencies pinned to exact versions. Name is always `"@figma/my-make-file"`. Contains both `dependencies` and `peerDependencies` for React 18 (peerDeps marked optional). Includes `pnpm.overrides` to pin Vite — npm ignores this field. |
| `vite.config.ts` | Vite 6 config. Registers `@vitejs/plugin-react` and `@tailwindcss/vite` (Tailwind v4 vite plugin). Both are required even if Tailwind is not actively used. Sets `@` alias to `./src`. Adds `assetsInclude` for `.svg` and `.csv` raw imports. |
| `postcss.config.mjs` | Intentionally empty. Tailwind v4's vite plugin handles PostCSS automatically; no `tailwindcss` entry needed here. |
| `README.md` | Brief description of the project generated by Figma Make. |
| `ATTRIBUTIONS.md` | Open-source license attributions for included packages. |
| `guidelines/Guidelines.md` | Project-specific usage guidelines written by the Figma Make session. |

---

## `src/main.tsx`

The React entry point. Calls `createRoot(document.getElementById("root")!)` and
renders `<App />`. Also imports `./styles/index.css` to load the CSS chain.

```tsx
import { createRoot } from "react-dom/client";
import App from "./app/App.tsx";
import "./styles/index.css";

createRoot(document.getElementById("root")!).render(<App />);
```

---

## `src/styles/`

| File | Purpose |
| ------ | --------- |
| `index.css` | CSS entry point. Imports `fonts.css`, `tailwind.css`, `theme.css` in that order. Order matters: theme variables must come after Tailwind resets. |
| `tailwind.css` | Contains `@import "tailwindcss"` — the Tailwind v4 entrypoint. |
| `fonts.css` | Font-face declarations or Google Fonts `@import` rules used by the design. |
| `theme.css` | CSS custom properties (`--color-*`, `--font-*`, spacing, etc.) derived from the Figma design tokens. This file is the bridge between Figma and Tailwind. Always take this file as-is from the export. |

---

## `src/app/`

The application shell. This is generic scaffolding that Figma Make writes
around the exported frames.

| File | Purpose |
| ------ | --------- |
| `App.tsx` | Root React component. Wraps everything in `<AppProvider>`, `<DndProvider backend={HTML5Backend}>`, and `<RouterProvider router={router}>`. |
| `routes.ts` | React Router v7 (`react-router`) route definitions using `createBrowserRouter`. Defines the root layout route and child content routes. |
| `context/AppContext.tsx` | Global application state context (theme, sidebar state, etc.). |
| `context/DocumentContext.tsx` | Document/content-level state context. |
| `data/documentStore.ts` | In-memory document store with CRUD operations. |
| `data/indexedDBService.ts` | IndexedDB persistence layer for document store. |
| `data/navigationStructure.ts` | Static navigation tree data for the app sidebar. |

## `src/app/components/`

### `ui/` — shadcn/radix-ui primitives

40+ component files, each a thin wrapper around a Radix UI primitive styled
with Tailwind and `class-variance-authority`. These are the standard shadcn/ui
components and are safe to treat as read-only library code.

Notable files: `accordion`, `alert-dialog`, `avatar`, `badge`, `breadcrumb`,
`button`, `calendar`, `card`, `carousel`, `chart`, `checkbox`, `collapsible`,
`command`, `context-menu`, `dialog`, `drawer`, `dropdown-menu`, `form`,
`hover-card`, `input`, `input-otp`, `label`, `menubar`, `navigation-menu`,
`pagination`, `popover`, `progress`, `radio-group`, `resizable`, `scroll-area`,
`select`, `separator`, `sheet`, `sidebar`, `skeleton`, `slider`, `sonner`,
`switch`, `table`, `tabs`, `textarea`, `toggle`, `toggle-group`, `tooltip`.

Support files: `use-mobile.ts` (viewport hook), `utils.ts` (`cn()` helper from
`clsx` + `tailwind-merge`).

### `figma/`

| File | Purpose |
| ------ | --------- |
| `ImageWithFallback.tsx` | `<img>` wrapper that catches load errors and displays a placeholder. Used by generated frame components that reference Figma-hosted images. |

### Custom components

These are generated by Figma Make to compose the `src/imports/` frames into a
navigable application:

| File | Typical role |
| ------ | ------------- |
| `Root.tsx` | Layout shell with sidebar + `<Outlet />` for nested routes. |
| `DocumentView.tsx` | Main content area; renders an imported Figma frame based on the active route. |
| `RightNavPanel.tsx` | Right-side navigation/details panel. |
| `FloatingMenu.tsx` | Floating action/context menu overlay. |
| `EditableContent.tsx` | Rich-text or structured content editor inlined in the view. |
| `ImageAnnotation.tsx` | Annotation overlay for image assets in the document. |

## `src/imports/`  ★ unique per export

This directory holds the actual Figma design output. Every file here is
auto-generated by Figma Make and changes between exports.

| Pattern | Purpose |
| --------- | --------- |
| `*.tsx` | One file per exported Figma frame. Renders the frame as a React component using Tailwind utility classes, inline styles, and shadcn/ui primitives. |
| `svg-<id>.ts` | Inline SVG data modules. Each exports a single `default` string containing a `data:image/svg+xml;base64,...` URI or raw SVG markup used by adjacent TSX files. |

**Rule**: Always replace the entire `src/imports/` directory from the ZIP when
applying an export. Never merge individual files — they form a coherent snapshot
of a single Figma session.

## Common integration pitfalls

1. **Missing `@` alias** — if merging into an existing `vite.config.ts` that
   lacks `resolve.alias`, all `@/...` imports will fail at build time.

2. **Tailwind v4 + config file conflict** — if a `tailwind.config.js` is
   present in the project root, Tailwind v4 may produce unexpected results.
   Remove it; Tailwind v4 reads configuration from CSS (`@theme` blocks in
   `theme.css`).

3. **Missing TypeScript and `@types`** — Figma Make exports do not include
   `typescript`, `@types/react`, `@types/react-dom`, or `@types/node` in
   `devDependencies`, and ship no `tsconfig.json`. Vite uses esbuild and runs
   fine without them, but IDE support and `tsc --noEmit` require them. Add:

   ```bash
   npm install --save-dev typescript@5 @types/react@18 @types/react-dom@18 @types/node
   ```

   And create `tsconfig.json` with `"moduleResolution": "bundler"`,
   `"allowImportingTsExtensions": true`, `"noEmit": true`, and the `@` path
   alias matching `vite.config.ts`.

4. **Known type errors in generated code** — observed in real exports; all are
   safe to `npm run dev` but fail strict `tsc`:
   - `saveTimeoutRef` used but never declared: add
     `const saveTimeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);`
   - Mouse handler typed for `<button>` on a `<div>`: widen parameter to
     `React.MouseEvent<Element>`.
   - `WebkitUserDrag` in inline style: cast as `React.CSSProperties`.
   - `href` on inner `<span>` inside `<a>`: remove the `href` from the `<span>`.

5. **peerDependencies double-install** — The Figma Make `package.json` lists
   React 18 in both `dependencies` and `peerDependencies`. Running `npm install`
   on this file directly works fine. The issue only arises if you hand-edit the
   file and remove one copy.

6. **`src/styles/index.css` import order** — fonts must be imported before
   tailwind (to avoid font class conflicts) and theme must come last (CSS
   variables override Tailwind defaults). The order is enforced by the file as
   exported; do not reorder.

7. **`src/imports/` frame references** — Figma frame TSX files often reference
   each other by relative path (`../imports/OtherFrame`). Partial replacement
   of files in `src/imports/` will break these cross-references.
